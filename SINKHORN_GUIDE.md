# Sinkhorn Topological Loss Implementation Guide

This document summarizes the changes and features added to the Topological Autoencoder (TAE) pipeline, focusing on the new Sinkhorn-based Optimal Transport (OT) topological loss.

## 0. Development Evolution Summary
| Feature | Original Plan (Planned) | Actual Implementation (Finalized) | Key Improvement |
| :--- | :--- | :--- | :--- |
| **Gradients** | Backprop through the loop | **Envelope Theorem** (.detach duals) | **~50x Memory Savings**, faster training |
| **Stability** | Exponential space Sinkhorn | **Log-Domain (logsumexp)** | No underflow, stable with small epsilon |
| **Scale** | Adaptive Weighting only | **Topo Multiplier (10^1~10^5)** | Prevented topo loss from being "masked" |
| **Search** | Manual tuning | **Random Multiplier Sweep** | Automated optimal parameter discovery |
| **GPU Sync** | .item() & in-place ops | **Zero-Sync / Masking Matrix** | Max GPU utilization, stable autograd |

## 1. Key Features

### 1.1 Optimized Sinkhorn Implementation
- **Log-Domain Computation:** Uses `logsumexp` for numerical stability, preventing underflow with small epsilon values.
- **Envelope Theorem Optimization:** Dual variables are detached after the Sinkhorn loop. Gradients are computed mathematically from the optimal potentials, saving significant GPU memory and accelerating training.
- **Non-in-place Masking:** Diagonal elements are zeroed using multiplication with a mask instead of in-place operations, ensuring full compatibility with PyTorch's autograd.
- **No GPU-CPU Sync:** Median calculations and constant initializations are performed entirely on the GPU to avoid performance bottlenecks.

### 1.2 Topo Multiplier & Hyperparameter Sweep
- **Magnitude Matching:** Standard Sinkhorn distances (~0.6) are much smaller than Reconstruction MSE (~10,000). We introduced a `--topo-multiplier` to scale the topological loss so its gradient is not masked.
- **Random Sweep:** `run_pipeline.sh` now performs a "Multiplier Sweep," testing 3 random scales between $10^1$ and $10^5$ for each dimension/metric combination.

## 2. Updated Scripts & Usage

### 2.1 `run_pipeline.sh`
Added support for skipping preprocessing and running the Sinkhorn sweep.
```bash
# Run full pipeline with Sinkhorn sweep
bash run_pipeline.sh

# Skip preprocessing (Step 1) and run training/vis
bash run_pipeline.sh --skip-preprocess
```

### 2.2 `TAE/training/train.py`
New arguments for Sinkhorn experimentation:
- `--sinkhorn`: Enables the Optimal Transport loss.
- `--topo-multiplier`: Sets the scaling factor for topological loss (default: 1.0).

### 2.3 `TAE/training/latent_vis.py`
Automatically detects and processes all models generated by the multiplier sweep.
- Output files include the multiplier in the name: `latent_{dim}d_sinkhorn_m{mult}.csv`
- UMAP plots show the multiplier in the title for easy comparison.

### 2.4 `TAE/training/visualization.py`
Aggregates logs from all sweep runs and compares them in final charts.
- **Learning Curves:** Generated for every unique `(dim, metric, multiplier)` triplet.
- **Comparison Charts:** Bar charts (Classifier accuracy, Best losses) now include `sink-m{mult}` labels to help identify the most effective scaling factor.

## 3. Mathematical Summary

The finalized loss function for Sinkhorn mode is:

$$L_{total} = \frac{1}{2\sigma_r^2} L_{recon} + \frac{1}{2\sigma_t^2} (Multiplier \cdot L_{topo}) + \log \sigma_r + \log \sigma_t$$

Where $L_{topo}$ is the debiased Sinkhorn Divergence:
$$L_{topo} = OT(P_{orig}, P_{lat}) - 0.5 \cdot OT(P_{lat}, P_{lat})$$

The $Multiplier$ allows the $L_{topo}$ gradient to compete with the $L_{recon}$ gradient, while the adaptive weights ($\sigma_r, \sigma_t$) fine-tune the balance during training.
